ASOBuilder

ASOBuilder es un proyecto que pretende agilizar el desarrollo de APIs generando código Java, el cual cumpla con las normativas y estándares requeridos por el cliente, liberando al desarrollador de trabajo monótono y repetitivo, permitiendo que este se enfoque solo en aquellos apartados que requieran supervisión humana.

Tecnologías:

Java 100%
Spring Boot 100%
Thymeleaf 100%

Propuesta y desarrollo: Miguel Villa

Año: 2019

Cliente: cliente no disponible

Problemática

Las APIs requeridas por el cliente se describen tanto en RAML como en archivos de Excel (insumos), en los cuales se indica tanto la presentación, datos de entrada y salida, así como el servicio el cual deben consumir (backend). Es a partir de estos insumos que el desarrollador deberá programar el API, sin embargo este proceso suele ser tardado y complejo, por una parte debido a la magnitud de entradas y salidas, y por otra parte debido a errores comunes que surgen durante el desarrollo, (errores de tipado, malas prácticas de programación).

Considerando la estructura del proyecto que conforma el API, y sabiendo que hay excepciones, del 100% de código que lo conforma, alrededor del 85% es código transcrito de los archivos RAML y Excel a código Java, 5% está relacionado con archivos de configuración y tan solo el 10% es código de mayor complejidad. Esto permite ver que existe una sección de los proyectos, que aunque necesaria, merma el tiempo de desarrollo.

Propuesta

Debido a la gran cantidad de APIs que ya se encuentran desarrolladas a partir de los insumos (Excel y RAML) y a sabiendas que el cambio de estos insumos impacta directamente al flujo que el cliente ya tiene definido para la construcción de sus APIs, se realizó la siguiente propuesta.

Desarrollar un programa de utilería que agilice el desarrollo de las APIs del cliente, generando código Java implementado sobre proyectos Maven, tomando como insumos de diseño del API los archivos RAML (.raml) y Excel (.xlsx), permitiendo disminuir hasta en un 90% el tiempo de desarrollo, así como reducir los errores comunes de desarrollo al mínimo. Brindándole a la empresa una herramienta para aprovechar al máximo el tiempo de sus recursos.

Resultados

ASOBuilder es un servicio web desarrollado con Spring Boot el cual realiza la lectura de archivos de diseño de un API; .xlsx y .raml para generar un proyecto maven comprimido en .zip, en cual contiene tanto los archivos de configuración del proyecto (.xml) como archivos .java.

En la figura 2 se muestra la estructura general del proyecto Maven resultante. De los tipos de clases que se generan destacan las denominadas como DTO y DTOInt por ser las clases más extensas, así como las clases MapStructMapper y ArchMapper que además de ser igual de extensas que las anteriores, son de mayor complejidad. Es importante mencionar que la imagen tan solo es una representación ya que en realidad la cantidad de clases generadas es mucho mayor y depende directamente de los insumos.

Conclusiones

En el siguiente gráfico se muestra una comparativa del tiempo empleado por un desarrollador y el tiempo empleado por el proyecto ASOBuilder; para el desarrollo de un API con 700 campos de entrada. Considerando el tiempo de desarrollo, atención a errores y despliegue; se hace evidente la diferencia de tiempo, ya que sumando el tiempo empleado por cada uno se obtiene un total de 13 días para el desarrollador y tan solo 2 días generando código ASOBuilder. A pesar de que ASOBuilder genera el código en cuestión de segundos, el tiempo marcado en el gráfico se debe a que el código generado no está completamente listo para ser desplegado, por lo que se requiere de un desarrollador para completar el código.

Trabajos futuros

Como se mencionó anteriormente; el desarrollo de esta utilería se hizo con base a los insumos que el cliente tiene definidos y tan solo se agregó un nuevo componente que genere el código, con la finalidad de no entorpecer el proceso que los desarrolladores tienen definido. Sin embargo, y sin entrar en más detalle sobre el proceso de diseño del API, se podría considerar cambiar dichos insumos ya que trabajar con archivos de excel resulta complicado y en ocasiones contraproducente más aún sabiendo que existen más y mejor opciones. Sin ir más lejos archivos como XML o JSON son opciones más simples y con más posibilidades.
